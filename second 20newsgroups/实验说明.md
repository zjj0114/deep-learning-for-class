# 实验说明

## 1. 任务

使用 PyTorch 对 20 Newsgroups 数据集中的两个类别做文本二分类：

- `alt.atheism`
- `soc.religion.christian`

当前项目优先使用根目录 `20_news_data.py` 对应的官方 train/test 划分；在此基础上，再从官方训练集里切出一小部分验证集做早停。

## 2. 模型选择

本作业在 `RNN / LSTM / GRU` 中选择了 `BiGRU`。

原因：

1. 普通 RNN 容易受到长序列梯度消失影响，通常不如 GRU 和 LSTM 稳定。
2. LSTM 表达能力强，但参数更多，训练开销更大。
3. GRU 在这个任务上通常能用更少参数达到接近甚至优于 LSTM 的效果。
4. 双向结构可以同时利用前后文，对新闻文本分类更有利。

因此，这里采用双向 GRU 作为精度和训练成本之间更稳妥的选择。

## 3. 数据处理

脚本默认对文本做以下预处理：

- 删除邮件头信息
- 删除明显的引用行
- 转小写
- 删除标点
- 删除数字
- 用空格分词

词表只根据训练集构建，避免测试集信息泄露。

## 4. 训练设置

- Embedding 维度：128
- GRU 隐层维度：128
- GRU 层数：2
- Dropout：0.35
- 优化器：AdamW
- 学习率：`7e-4`
- 损失函数：CrossEntropyLoss
- 早停策略：验证集精度连续若干轮不提升则停止

## 5. 结果解读

训练结束后重点看：

- `classification_report.txt`
- `metrics.json`
- `training_history.png`
- `confusion_matrix.png`

如果验证集和测试集准确率比较接近，说明模型泛化较稳定；如果训练准确率明显高于验证准确率，则说明过拟合，需要减小模型规模或增强正则化。

## 6. 一句结论

对于这次二分类新闻文本任务，BiGRU 一般比普通 RNN 更稳定，训练速度又通常快于 LSTM，是一个比较适合交作业的选择。
